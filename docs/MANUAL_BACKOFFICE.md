# คู่มือการใช้งานหลังบ้าน

คู่มือนี้สำหรับเจ้าหน้าที่ ผู้ดูแลระบบ หัวหน้างาน และผู้ที่เกี่ยวข้องกับการบริหารจัดการเคสในระบบ HealthHelp

## หน้าที่ใช้งาน
- `/admin/login`
- `/admin/dashboard`
- `/admin/create`
- `/admin/cases`
- `/admin/master-data`
- `/admin/settings`
- `/admin/users`

## Task 1: เข้าสู่ระบบเจ้าหน้าที่
- [ ] เปิดหน้า `/admin/login`
- [ ] กรอกอีเมลและรหัสผ่าน
- [ ] กดปุ่ม `เข้าสู่ระบบ`
- [ ] ตรวจสอบว่าระบบพาเข้า `/admin/dashboard`

## Task 2: ตรวจสอบภาพรวมจากแดชบอร์ด
- [ ] เปิดหน้า `/admin/dashboard`
- [ ] ตรวจสอบจำนวนเคสรวม เคสเปิด เคสกำลังดำเนินการ และเคสที่ปิดแล้ว
- [ ] ดูกราฟแนวโน้มเคสตามช่วงเวลา
- [ ] ดูบล็อก `ผลงานเจ้าหน้าที่`
- [ ] ใช้ตัวกรองช่วงเวลา เช่น รายวัน รายเดือน หรือรายปี หากต้องการ

## Task 3: สร้างเคสใหม่จากหลังบ้าน
- [ ] เปิดหน้า `/admin/create`
- [ ] กรอกข้อมูลเคสใหม่แทนผู้แจ้ง
- [ ] ตรวจสอบหมวดหมู่ รายละเอียด และข้อมูลติดต่อผู้แจ้ง
- [ ] แนบไฟล์ หากมี
- [ ] กดบันทึกเพื่อสร้างเคสใหม่

## Task 4: ดูรายการเคสทั้งหมด
- [ ] เปิดหน้า `/admin/cases`
- [ ] ใช้ตัวกรองสถานะ ความเร่งด่วน หรือคำค้นหา
- [ ] ตรวจสอบรายการเคสตามผลลัพธ์ที่ระบบแสดง
- [ ] เปิดเคสที่ต้องการเพื่อเข้าไปดูรายละเอียด

## Task 5: เปิดดูรายละเอียดเคส
- [ ] คลิกเคสจากหน้ารายการเคส
- [ ] ตรวจสอบข้อมูลผู้แจ้ง รายละเอียดปัญหา สถานะ ความเร่งด่วน และผู้รับผิดชอบ
- [ ] ตรวจสอบไทม์ไลน์กิจกรรมและประวัติการสนทนา

## Task 6: บันทึกหมายเหตุภายใน
- [ ] เปิดหน้ารายละเอียดเคส
- [ ] ใช้ช่อง `บันทึกภายใน` หรือส่วนที่เกี่ยวข้องกับการทำงานภายในทีม
- [ ] กรอกข้อความที่ต้องการบันทึก
- [ ] แนบไฟล์ภายใน หากมี
- [ ] กดบันทึก
- [ ] ตรวจสอบว่าโน้ตปรากฏใน timeline ฝั่งเจ้าหน้าที่

## Task 7: ตอบกลับผู้แจ้งผ่านระบบ
- [ ] เปิดหน้ารายละเอียดเคส
- [ ] ใช้ช่องตอบกลับผู้แจ้ง
- [ ] พิมพ์ข้อความตอบกลับ
- [ ] แนบไฟล์หรือรูปภาพเพิ่มเติม หากมี
- [ ] ตรวจสอบ preview ก่อนส่ง
- [ ] กดส่งข้อความ
- [ ] ตรวจสอบว่าข้อความปรากฏในไทม์ไลน์ทั้งฝั่งเจ้าหน้าที่และผู้แจ้ง

## Task 8: มอบหมายผู้รับผิดชอบ
- [ ] เปิดหน้ารายละเอียดเคส
- [ ] เลือกผู้รับผิดชอบจากรายชื่อเจ้าหน้าที่
- [ ] กดบันทึกการมอบหมาย
- [ ] ตรวจสอบว่าชื่อผู้รับผิดชอบอัปเดตในหน้าเคสและแดชบอร์ด

## Task 9: เปลี่ยนสถานะเคส
- [ ] เปิดหน้ารายละเอียดเคส
- [ ] เลือกสถานะใหม่ เช่น `เปิด`, `กำลังดำเนินการ`, `รอข้อมูลเพิ่มเติม`, `แก้ไขแล้ว`, `ปิดเคส`
- [ ] กดบันทึก
- [ ] ตรวจสอบว่า timeline และสถานะล่าสุดอัปเดตถูกต้อง

## Task 10: ลบไฟล์แนบ
- [ ] เปิดหน้ารายละเอียดเคส
- [ ] เลื่อนหาไฟล์แนบในไทม์ไลน์
- [ ] กดไอคอนถังขยะที่ไฟล์นั้น
- [ ] อ่านกล่องยืนยันการลบ
- [ ] กด `ลบไฟล์`
- [ ] ตรวจสอบว่าไฟล์ถูกลบออกจากหน้าเว็บ โฟลเดอร์เก็บไฟล์ และชีต `Attachments`

## Task 11: นำเข้าเคสจาก Excel หรือ CSV
- [ ] เปิดเมนูนำเข้าเคสจากหน้าหลังบ้าน
- [ ] เตรียมไฟล์ `.xlsx` หรือ `.csv`
- [ ] ตรวจสอบลำดับคอลัมน์ให้ตรงกับที่ระบบรองรับ
- [ ] `ปัญหา | รายละเอียด | ชื่อผู้แจ้ง | เบอร์โทร | อีเมล | Line ID | หมวดหมู่`
- [ ] อัปโหลดไฟล์เข้าสู่ระบบ
- [ ] ตรวจสอบ preview และแถวที่มี error
- [ ] กดปุ่มยืนยันนำเข้าเมื่อข้อมูลถูกต้อง
- [ ] ตรวจสอบจำนวนรายการที่นำเข้าสำเร็จ

## Task 12: จัดการฐานข้อมูลหน่วยงาน/โรงพยาบาล
- [ ] เปิดหน้า `/admin/master-data`
- [ ] เตรียมไฟล์ CSV สำหรับข้อมูลโรงพยาบาลหรือหน่วยงาน
- [ ] อัปโหลดไฟล์เข้าสู่ระบบ
- [ ] ตรวจสอบผลการนำเข้า ว่ามีจำนวนสำเร็จและผิดพลาดเท่าใด
- [ ] ค้นหาข้อมูลหน่วยงานเพื่อยืนยันว่าข้อมูลถูกอัปเดตแล้ว

## Task 13: จัดการหมวดหมู่ปัญหา
- [ ] เปิดหน้า `/admin/settings`
- [ ] เลือกแท็บ `หมวดหมู่`
- [ ] เพิ่มหมวดหมู่ใหม่ พร้อมเลือกระดับความเร่งด่วนเริ่มต้น
- [ ] เปิดใช้งานหรือปิดใช้งานหมวดหมู่ตามความจำเป็น

## Task 14: ตั้งค่า SLA
- [ ] เปิดหน้า `/admin/settings`
- [ ] เลือกแท็บ `กฎ SLA`
- [ ] ปรับจำนวนชั่วโมงในการแก้ไขตามระดับความเร่งด่วน
- [ ] ปรับจำนวนชั่วโมงสำหรับแจ้งเตือนล่วงหน้า
- [ ] กดบันทึกในแต่ละกฎที่มีการแก้ไข

## Task 15: ตรวจสอบผู้ใช้งานระบบ
- [ ] เปิดหน้า `/admin/users`
- [ ] ตรวจสอบรายชื่อเจ้าหน้าที่ อีเมล บทบาท และสถานะการใช้งาน
- [ ] ใช้หน้านี้เพื่อตรวจสอบสิทธิ์เบื้องต้นของผู้ใช้ในระบบ

## Task 16: เปิดดูข้อมูลใน Google Sheet
- [ ] เปิดเมนู `Sheet ข้อมูล` จากแถบด้านข้าง
- [ ] ตรวจสอบชีตหลัก `เคสที่แจ้งมาแล้ว`
- [ ] ตรวจสอบชีต `Attachments` สำหรับไฟล์แนบ
- [ ] ใช้ชีตนี้เพื่อทวนสอบข้อมูลหรือเป็นหลักฐานประกอบการทำงาน

## สิทธิ์การใช้งาน

### Admin
- [ ] เข้าถึงทุกเมนู
- [ ] ดูผู้ใช้งานระบบได้
- [ ] จัดการตั้งค่าระบบได้
- [ ] จัดการข้อมูลหลักและนำเข้าข้อมูลได้
- [ ] ลบไฟล์แนบได้

### Supervisor
- [ ] เข้าถึงแดชบอร์ด รายการเคส ตั้งค่าระบบ และฐานข้อมูลได้
- [ ] มอบหมายงานและติดตามทีมได้
- [ ] นำเข้าข้อมูลและลบไฟล์แนบได้
- [ ] ไม่สามารถเข้าหน้า `ผู้ใช้งานระบบ` ได้

### Staff
- [ ] เข้าถึงแดชบอร์ด สร้างเคสใหม่ และจัดการเคสได้
- [ ] ตอบกลับผู้แจ้งและอัปเดตสถานะได้
- [ ] ไม่สามารถเข้าตั้งค่าระบบหรือข้อมูลหลักได้

### Viewer
- [ ] เข้าดูแดชบอร์ดและรายการเคสได้
- [ ] ใช้งานแบบติดตามหรือดูข้อมูลเป็นหลัก
- [ ] ไม่ควรใช้สำหรับแก้ไขข้อมูลสำคัญ

## หมายเหตุสำหรับใช้งานจริง
- [ ] ควรสำรองข้อมูลก่อนรันสคริปต์ sync หรือ import จำนวนมาก
- [ ] ควรตรวจสอบข้อมูลใน Google Sheets หลังมีการอัปโหลดไฟล์หรือแก้ไขข้อมูลสำคัญ
- [ ] หากใช้งานจากหลายเครื่อง ให้ตรวจสอบว่าเครื่อง server และฐานข้อมูลเชื่อมต่อได้ตามปกติ
- [ ] ก่อนนำขึ้น production ควรตรวจสอบ `.env`, สิทธิ์ผู้ใช้, Google Sheets credentials และพื้นที่เก็บไฟล์แนบ
